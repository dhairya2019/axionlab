<task type="auto">
  <name>Task 2: Update config files for Next.js</name>
  <files>vercel.json, tsconfig.json, postcss.config.mjs, next.config.mjs</files>
  <action>
**vercel.json** — Replace entire contents with:
```json
{
  "version": 2
}
```
Remove ALL of: "framework", "buildCommand", "outputDirectory", "rewrites". Vercel auto-detects Next.js from next.config.mjs. The SPA catch-all rewrite (`/(.*) -> /index.html`) MUST be removed — it conflicts with Next.js page routing.

**tsconfig.json** — Replace entire contents with:
```json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["DOM", "DOM.Iterable", "ESNext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [
      {
        "name": "next"
      }
    ],
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```
Key changes from current: `jsx: "preserve"` (not "react-jsx"), `moduleResolution: "bundler"`, `plugins: [{ name: "next" }]`, `paths: { "@/*": ["./*"] }`, `include` adds `next-env.d.ts` and `.next/types/**/*.ts`.

**postcss.config.mjs** — Create new file (delete postcss.config.js if it exists). Contents:
```javascript
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
export default config;
```
Use `.mjs` extension because project has `"type": "module"`. Do NOT include autoprefixer — Tailwind v4 handles vendor prefixes internally.

If `postcss.config.js` exists (old file), delete it.

**next.config.mjs** — Replace entire contents with:
```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {};

export default nextConfig;
```
Remove `images.unoptimized: true` — Vercel handles image optimization natively. Remove any other Vite-era settings. Keep it minimal — MDX config will be added in Phase 2.

Do NOT add `output: 'export'` — this would disable API Route Handlers which are needed for chat and email.
