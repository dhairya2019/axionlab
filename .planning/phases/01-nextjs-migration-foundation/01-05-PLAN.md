**Expected:** Build completes with zero errors. All 7 pages + not-found + 2 API routes compile.

**If build errors occur**, diagnose and fix them immediately. Common issues to watch for:

1. **"You're importing a component that needs useState"** — A page is importing from framer-motion directly instead of MotionWrapper. Fix: change the import.

2. **"Module not found: @/components/..."** — Path alias not resolving. Fix: check tsconfig.json has paths configured.

3. **"Cannot find module 'next'"** — npm install did not complete. Fix: run `npm install`.

4. **Type errors with @types/react** — React 19 types may differ from 18. Fix: update the specific type usage.

5. **"Export default is not defined"** — A page.tsx file may be missing its default export. Fix: ensure every page.tsx has `export default function PageName()`.

6. **PostCSS/Tailwind errors** — @tailwindcss/postcss may report issues with globals.css. Fix: check the @import and @theme syntax.

Fix each error as it appears. Re-run `next build` after each fix until it succeeds. Do NOT add `// @ts-ignore` or `// eslint-disable` to suppress errors — fix the root cause.

After successful build, verify the `.next/` directory was created:

test -d .next && echo "BUILD OUTPUT EXISTS"
