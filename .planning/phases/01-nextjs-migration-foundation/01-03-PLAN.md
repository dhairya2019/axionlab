<objective>
Migrate both API endpoints from Vercel serverless function format (handler(req, res)) to Next.js App Router Route Handler format (export async function POST(request: Request)). The business logic (Gemini AI calls, SendGrid email dispatch) stays identical — only the HTTP interface changes.

Purpose: Without this migration, the chatbot widget and contact form will return 500 errors because the old handler(req, res) signature is incompatible with Route Handlers. This plan runs in parallel with Plan 02 (both Wave 2, no file overlap).

Output: Two Route Handler files at app/api/chat/route.ts and app/api/send-email/route.ts that respond to the same /api/chat and /api/send-email URLs the frontend already fetches.
</objective>

Specific changes:

1. **Function signature:**
   - Before: `export default async function handler(req, res) {`
   - After: `export async function POST(request: Request) {`

2. **Method check removal:**
   - Remove the `if (req.method !== 'POST')` block entirely — named `POST` export only responds to POST.

3. **Body parsing:**
   - Before: `const { message, history } = req.body;`
   - After: `const { message, history } = await request.json();`

4. **Response construction (EVERY occurrence):**
   - Before: `return res.status(200).json({ text: ..., isSystem: ... });`
   - After: `return Response.json({ text: ..., isSystem: ... });`
   - Before: `return res.status(500).json({ error: ... });`
   - After: `return Response.json({ error: ... }, { status: 500 });`
   - Before: `return res.status(405).json({ error: ... });`
   - After: Remove entirely (POST-only by convention).

5. **Gemini model update:**
   - Before: Hard-coded `'gemini-3-flash-preview'` (unstable preview model)
   - After: `process.env.GEMINI_MODEL ?? 'gemini-2.0-flash'`
   - This makes the model configurable via environment variable with a stable default.

6. **TypeScript:** Add type annotation for the function parameter: `request: Request`. The rest of the business logic is JavaScript-compatible TypeScript (no additional types needed for the Gemini/SendGrid calls).

7. **Imports:** Keep all existing imports (GoogleGenAI, Type from @google/genai, sgMail from @sendgrid/mail). Do NOT import NextResponse — use the Web standard `Response.json()` which is available natively in Route Handlers.

Preserve ALL of the following from the original chat.js without modification:
