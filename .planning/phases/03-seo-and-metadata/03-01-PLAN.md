**app/layout.tsx:** Update the existing `metadata` export. Keep all other code (imports, font config, RootLayout component) unchanged. Replace the current metadata object with:

```typescript
export const metadata: Metadata = {
  metadataBase: new URL('https://axionlab.in'),
  title: {
    template: '%s | AXIONLAB',
    default: 'AXIONLAB | Engineering for the obsessed.',
  },
  description: 'Independent systems engineering lab designing commerce infrastructure and high-performance applications.',
  openGraph: {
    siteName: 'AXIONLAB',
    type: 'website',
    locale: 'en_US',
  },
  twitter: {
    card: 'summary_large_image',
  },
}
```

Key points:
- `metadataBase` resolves all relative URLs in OG fields to absolute
- `title.template` means child pages only set `title: 'Philosophy'` and get "Philosophy | AXIONLAB"
- `title.default` is used when no child page sets a title (e.g., home page fallback — but home will get its own in Plan 03-02)
- `openGraph.siteName` and `twitter.card` are base fields inherited by pages that do NOT define their own `openGraph`/`twitter` objects. However, pages that DO define `openGraph` must include all needed fields (shallow merge replaces entire nested object).

**app/robots.ts:** Create new file:

```typescript
import type { MetadataRoute } from 'next'

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: '*',
      allow: '/',
    },
    sitemap: 'https://axionlab.in/sitemap.xml',
  }
}
```

This generates /robots.txt at build time allowing all crawlers and pointing to the sitemap.
