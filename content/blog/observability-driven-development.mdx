## Instrument First, Build Second

The conventional approach — build features, then add monitoring — creates blind spots. By the time you add observability, the system's failure modes are already invisible.

### Structured Logging

Every log entry should be machine-parseable:

```typescript
logger.info('request_processed', {
  duration_ms: 42,
  status: 200,
  endpoint: '/api/orders',
  trace_id: context.traceId,
})
```

### Metrics That Matter

Not all metrics are equal. Focus on the four golden signals:

- **Latency**: Time to serve a request
- **Traffic**: Demand on the system
- **Errors**: Rate of failed requests
- **Saturation**: How full the system is

> You cannot improve what you cannot measure. But measuring everything is just noise.

### Distributed Tracing

In a microservices architecture, a single user request touches dozens of services. Without tracing, debugging is archaeology.
